\begin{frame}{Datasets}
Os principais datasets para a tarefa de detecção de objetos são:
\begin{itemize}
\item COCO Dataset
\item PASCAL VOC dataset
\end{itemize}
\end{frame}

%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
\subsection{COCO Dataset}
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%

\begin{frame}{Datasets}{COCO Dataset}
\begin{itemize}
\item O {\bf Microsoft COCO Dataset} ({\it Common Objects in Context})  é um conjunto de dados de detecção e segmentação de objetos publicado pela Microsoft. 
\item Possui 330K imagens (200K+ anotadas).
\item Possui >2M instâncias em 80 classes.
\item Utiliza o formato JSON para as anotações.
\end{itemize}
\end{frame}


%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
\subsection{Pascal VOC Dataset}
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%

\begin{frame}{Datasets}{PASCAL VOC Dataset}
\begin{itemize}
\item O {\bf PASCAL VOC Dataset} fornece um dataset de imagens padronizado para detecção de objetos.
\item Possui $\approx$11K imagens contendo $\approx$ 27K objetos anotados pertencentes à 20 classes.
\item Utiliza o formato XML para as anotações.
\end{itemize}
\end{frame}

\section{Annotations Format}
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%

\begin{frame}{Annotations Format}
Os principais formatos para anotação são definidos pelos dois datasets:
\begin{itemize}
\item {\bf Pascal VOC Dataset}
\item {\bf COCO Dataset}
\end{itemize}
\end{frame}


%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
\subsection{COCO Format}
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%

\begin{frame}{Datasets}{COCO Dataset Format}
O formato {\bf COCO Dataset} utiliza um arquivo {\bf JSON} de anotação, o qual possui os seguintes blocos:
\begin{itemize}
\item {\bf info}: contém informações sobre o dataset.
\item {\bf licenses}: contém uma lista de licenças das imagens.
\item {\bf categories}: contém uma lista de categorias.
\item {\bf images}: contém informações de todas as imagens, exceto bounding boxes.
\item {\bf annotations}: contém lista dos {\it bounding boxes}.
\end{itemize}
\end{frame}

%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
\subsection{Pascal VOC Format}
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%

\begin{frame}{Datasets}{Pascal VOC Dataset}
O formato {\bf Pascal VOC Dataset} utiliza um arquivo {\bf XML} de anotação, o qual possui os seguintes blocos:
\begin{itemize}
\item {\bf Folder}: nome da pasta que contém o arquivo.
\item {\bf Filename}: nome do arquivo existente na pasta.
\item {\bf Size}: informações sobre as dimensões da imagem.
\item {\bf Object}: informações sobre o objeto a ser identificado.
\end{itemize}
\end{frame}

%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%

\begin{frame}{Datasets}{Pascal VOC Dataset}
As informações sobre o objetos são:
\begin{itemize}
\item {\bf Name}: nome do objeto a ser identificado.
\item {\bf Truncated}: informa se o objeto é visível parcialmente na imagem.
\item {\bf Dificult}: indica se o objeto é difícil de detectar.
\item {\bf Bounding box}: coordenadas do {\it bounding boxe}.
\end{itemize}
\end{frame}

%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%

\begin{frame}{Datasets}{Pascal VOC vs COCO Dataset Format}
As principais diferenças entre os formatos são:
\begin{itemize}
\item Pascal VOC usa anotações no formato XML, um arquivo XML por imagem.
\item COCO Dataset utiliza um único arquivo JSON para armazenar as anotações.
\end{itemize}
\end{frame}

%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
\subsection{Albumentations Format}
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%

\begin{frame}[fragile]{Albumentations Format}
\begin{itemize}
\item O formato Albumentations é similar ao Pascal VOC.
\item Difere em relação às coordenadas do bounding boxes, pois utiliza valores normalizados.
\item Para normalizar os valores, divide-se as coordenadas em pixels dos eixos x e y pela altura e largura da imagem.
\end{itemize}

\vspace{2cm}
\tiny{Fonte: \url{https://albumentations.ai/docs/getting_started/bounding_boxes_augmentation/}}
\end{frame}

%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
\subsection{YOLO Format}
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%

\begin{frame}[fragile]{YOLO Format}
\begin{itemize}
\item O formato YOLO utiliza um arquivo TXT.
\begin{itemize}
\item YOLO refere-se ao modelo de mesmo nome.
\end{itemize}
\item Cada arquito .TXT possui anotações referentes a uma imagem.
\item As anotações possuem o seguinte formato:
\begin{verbatim}
<object-class> <x> <y> <width> <height>
\end{verbatim}
\item Para cada objeto presente na imagem, um linha é criada.
\begin{verbatim}
0 45 55 29 67
1 99 83 28 44
\end{verbatim}
\end{itemize}
\end{frame}
